# Wiki 健康检查报告

> 本模板定义 lint 操作的输出格式。agent 按此结构输出报告，确保每次运行格式一致。

## 报告头

```markdown
# Wiki 健康检查报告

- **检查时间**：{{ISO 日期}}
- **Wiki 版本**：revision {{n}}
- **页面总数**：{{n}}
```

## 检查结果

每个维度使用统一格式：

```markdown
### {{维度名称}}

**状态**：✅ 无问题 / ⚠️ 发现 N 个问题 / ℹ️ 无法检查（说明原因）

{{如有问题，列出具体发现：}}
- {{问题描述}}（涉及 [[页面名]]）
```

### 八个维度（按此顺序输出）

1. **一致性** — 不同页面之间是否有矛盾的说法
2. **孤立页面** — 哪些页面没有被任何其他页面引用（入站链接为 0）
3. **缺失页面** — 正文中提到的 `[[双链]]` 是否都有对应的实际文件
4. **数据缺口** — 哪些重要模块/功能/流程还没有被文档化
5. **过时信息** — frontmatter 中的 `source` 路径是否还指向有效文件
6. **交叉引用** — 有关系的页面是否互相引用了对方
7. **归属完整性** — 每个 feature 页面的 `module` 字段是否都指向了存在的模块页面
8. **层级正确性** — 页面内容是否匹配其所在层级

```markdown
## 已执行的修复

- {{修复操作描述}}（影响 [[页面名]]）
```
