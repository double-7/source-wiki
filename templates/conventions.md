## 项目结构约定

### 目录职责

```
src/
├── controllers/    # 请求处理层
├── services/       # 业务逻辑层
├── repositories/   # 数据访问层
├── models/         # 数据模型定义
├── utils/          # 工具函数
└── config/         # 配置文件
```

> 根据项目实际结构描述每个目录的职责和约定。
> 标注是否有严格的分层约束（如 controller 不能直接访问 repository）。
